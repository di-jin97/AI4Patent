# 专利分析技能套件

基于 opencode 平台的一套 AI 技能，覆盖专利分析常见场景：分类标引、深度剖析、侵权检索、PCT 评审、Idea 预审。

---

## patent-classifier — 分类标引

**功能**：将一批专利按技术标签体系进行分类标引，输出结构化 Excel 报告。

**输入**：专利清单（Excel/CSV），至少包含专利号。如有标题、摘要可提高分类准确度。标签体系可使用内置的存储领域分类（8 个一级类），也可由用户自定义。

**输出**：Excel 文件，每件专利标注一级分类、二级分类、置信度和分类理由。同时输出分类分布概览。

---

## storage-patent-deepdive — 存储专利深度剖析

**功能**：对一批存储领域专利进行系统分析，涵盖分类标引、价值评分、Claim 1 要件分解、竞争格局分析、技术趋势研判，输出 Word 报告和配套 Excel 底表。

**输入**：专利清单 Excel，至少包含专利号。有标题、摘要、申请人、权利要求时分析更充分。可选参数为技术细分方向（如不指定则由 AI 自动判断）。

**输出**：
- **Word 报告**（15-20 页）：技术分类全景、竞争格局（技术路线对比矩阵、申请人画像）、重点专利深度剖析、技术趋势与布局建议。
- **Excel 底表**：包含全部专利的逐件分析——技术方案摘要、Claim 1 要件级分解（F1/F2/F3…，标注特征类型和解决的问题）、综合评分与价值等级、分类分布统计。

---

## seek-cfp-patent — 专利侵权检索

**功能**：解析专利权利要求，提取发明点，搜索目标友商产品与技术文档，进行逐特征加权比对，判断侵权风险。

**输入**：专利清单，必须包含第一权利要求（Claim 1 全文）。有标题、摘要、背景技术更佳。可选参数包括目标友商列表和匹配阈值（默认 50%）。内置 15 家存储领域友商（含海外与中国厂商）。

**输出**：Excel 文件，每条记录为一个专利与一个友商的比对结果，包含匹配率、判定结果（匹配/部分匹配/不匹配）、发明点命中情况、关键证据原文片段及来源 URL、建议行动项。对话同步输出简要比对摘要。

---

## patent-pct-review — PCT 申请评审

**功能**：评审一批专利是否需要进入 PCT 阶段，通过五个维度打分排序，辅助决策。

**输入**：两个文件——Excel（含 Priority sheet，专利号在 Patent Ref 列）和当前目录下的 DOCX 专利申请文件。AI 按专利号前 8 位数字自动匹配文件。

**输出**：原 Excel 副本，新增六列——发明概述、市场价值、创新性、取证手段、可规避性、PCT 建议（进 PCT 或不进 PCT）。进入 PCT 比例约 40%，按总分降序排列。

---

## patent-IDEA-analyzer — Idea 胶片预审

**功能**：在专利正式撰写前，对 Idea 胶片进行撰写质量检查与创新性分析，输出预审报告。

**输入**：PPT 或 Word 格式的专利 Idea 胶片（技术交底书雏形文档）。

**输出**：预审报告（Markdown 格式，可按需转为 Word），包含：
- **创新性分析**：三步法提炼发明点、检索现有技术进行对比、评估创造性、保护点重写建议、实施例扩展方向。
- **质量检查**：六项指标逐一审核——技术问题清晰度、技术方案完整性、技术效果对应性、保护点合理性、实施例充分性、附图标号规范性。每项给出判定和具体改进意见。

## 使用说明

将技能目录放置于 `~/.config/opencode/skills/` 下，启动 opencode 后，AI 会根据用户指令自动匹配对应技能，也可手动加载：`load skill <技能名>`。

## 依赖配置

专利检索和网页抓取依赖 Exa 搜索工具。在 `opencode.json` 中添加以下配置：

```json
{
  "mcpServers": {
    "exa": {
      "type": "remote",
      "url": "https://mcp.exa.ai/mcp"
    }
  }
}
```

若无 Exa，技能仍可运行，但外部信息检索功能受限，分类和侵权分析的准确度会下降。
