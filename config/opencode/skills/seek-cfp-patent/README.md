# Seek CFP Patent

解析专利权利要求，提取发明点，搜索目标友商产品与技术文档，进行逐特征加权比对，判断侵权风险。

---

## 是什么

上传一份含第一权利要求的专利清单，AI 自动拆分 Claim 1 为独立技术特征，标注发明点，然后搜索目标友商的产品文档和技术资料，逐特征比对并计算加权匹配率，输出侵权分析 Excel 报告。内置 15 家存储领域友商（含海外与中国厂商），支持自定义友商列表。

适用领域：存储（SSD、HDD、分布式存储、存储控制器、NVMe 协议等）。

## 使用方式

提供含 Claim 1 的专利清单后，触发示例：

- "帮我做侵权检索"
- "分析这些专利有没有侵权风险"
- "做 claim chart 分析"
- "对比这些专利和友商产品"

输入要求：
| 字段 | 必需 | 说明 |
|------|------|------|
| 专利号 | ✓ | 如 US12334926B2 |
| 标题 | ✓ | 专利标题 |
| 摘要 | ✓ | 专利摘要 |
| 第一权利要求 | ✓ | Claim 1 全文 |
| 背景技术/技术问题/有益效果 | 可选 | 用于发明点提取，建议提供 |

可选覆盖：目标友商列表（默认 Gate 后 ≤3 家）、匹配阈值（默认 50%）。

---

## 输出说明

输出一份 Excel 文件，每条记录为一个专利与一个匹配友商的结果（匹配率 <50% 不写入）：

| 列名 | 说明 |
|------|------|
| 专利号 | 专利号 |
| 标题 | 专利标题 |
| 技术分类 | 中文显示，如"数据保护" |
| 发明点摘要 | 1-2 句概括核心发明点 |
| 友商 | 如 Samsung |
| 产品名称 | 具体型号，如 PM9A3 |
| 匹配率 | 如 85% |
| 判定 | 匹配 / 部分匹配 |
| 发明点命中 | 如 F2✅ F3✅(DOE) |
| 相似点说明 | 功能等价和效果一致的推断 |
| 关键证据片段 | 厂商文档原文（≤80字） |
| 证据 URL | 来源链接 |
| 建议行动 | 具体到文档名/章节的跟进建议 |

## 注意事项

- 发明点特征若全部 ❌，该友商结果不输出，无论匹配率多高
- 匹配率 <50% 的友商结果不输出（对话不展示，Excel 不写入）
- 每个 ✅/⚠️ 必须有原文证据片段（≤80字）+ URL；找不到原文的 ✅ 必须降级为 ⚠️
- 中文专利不得直接用中文搜索海外友商，须先建术语对照表
- 批量处理（≥3 条专利）时建议按 3 条一批拆分并行运行，防止上下文溢出
