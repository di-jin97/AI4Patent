# Storage Patent Deepdive — 存储领域专利深度剖析与竞争态势分析（精细化版）

对一批存储领域专利做系统性精细分析：自动补全数据 → 分类标引 → 每件专利技术方案一句话摘要 →
价值评分排序 → 重点专利深度Claim 1要件级分解 → 分类内技术方案横向对比 →
竞争格局深度分析（含技术路线对比矩阵+申请人画像） → 技术趋势与布局建议 → 按需输出 Word / PPT。

---

## Step 0 — 询问输出范围与格式

启动后先问用户：

```
需要什么输出格式？
A. Word 报告（必选）— 竞争格局 + 创新洞察 + 布局建议（适合内部研究/归档）
B. PPT 汇报   （可选）— 约10页可视化版本（适合给领导汇报）
C. 两者都要
```

问完后输出本次分析范围概览，请用户确认后再继续。

---

## Step 0.5 — 询问技术领域细分方向

启动分类前，先问用户：

```
这批专利属于哪个技术领域或细分方向？
例如：SSD介质、分布式存储、NVMe协议、存储芯片、文件系统、存储管理……
```

用户回答后，进入 Phase 1。

# Phase 1 — 数据准备与分类

## 1.1 输入解析与摘要浏览

读取 `.xlsx` / `.csv` / 内联表格，字段映射（中英文容差）：

| 标准字段 | 常见变体 |
|---------|---------|
| 专利号 | 公开号、Publication Number、Patent No. |
| 标题 | Title、发明名称 |
| 摘要 | Abstract、说明书摘要 |
| 申请人 | Applicant、Assignee、专利权人 |
| 第一权利要求 | Claim 1、独立权利要求 |
| 同族数量 | Family Size、同族数 |
| 布局国家 | Countries、地域、Jurisdictions |
| 技术问题/手段/功效 | 可选，有则直接使用 |

输出完整性概览（非阻塞）：
```
总专利数: N | 已有摘要: X 篇 | 已有 Claim 1: Y 篇 | 待抓取补全: Z 篇
```
仅当专利号完全缺失时阻塞。

**浏览完所有专利的标题和摘要**（已有的数据），理解这批专利的技术内容分布。

输出初步观察：
```
领域: {用户指定的方向，例如 SSD 介质}
专利数: N
初步观察: 
- 涉及的技术方向主要有：{方向A}、{方向B}、{方向C}……
- 主流申请人：{申请人列表}
```

## 1.2 Google Patents 自动补全

**只要有专利号即可启动**，标题/摘要/Claim 1 任一缺失自动触发。

使用 `exa_web_fetch_exa` 抓取 Google Patents 页面：

```
exa_web_fetch_exa(urls=["https://patents.google.com/patent/{专利号}/en"], maxCharacters=100000)
```

提取：标题、摘要、申请人、公开日、IPC/CPC、Claim 1、同族数、布局国家。不抓说明书全文。

**4 项校验**：标题非空（>5字）+ 摘要非空（>20字）+ Claim 1 非空 + 专利号与页面匹配。
不通过则标"数据不全"，后续跳过方案提炼，仅做浅层评分。

抓取后输出：`成功 A 篇 / 数据不全 B 篇 / 失败 C 篇`

## 1.3 动态分类标准制定与分类

### 1.3.1 浏览所有专利，制定分类标准

基于以下输入，制定分类标准：
- 用户指定的技术领域（Step 0.5）
- 所有专利的标题和摘要内容

**你的任务是：**

1. **浏览全部专利的标题和摘要**，理解各专利的技术主题
2. 根据专利的实际技术内容，**制定一套分类体系**（3-8 个分类标签）
3. 分类标签应当是**该细分领域内有意义的技术方向划分**
4. 每个分类给出简要说明

### 1.3.2 提交分类标准给用户审批

输出以下格式，等待用户确认：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
基于 {用户指定的领域} 专利内容分析，建议以下分类标准：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. {分类A名称} — {简要说明}
2. {分类B名称} — {简要说明}
...

是否同意以上分类标准？如果同意，我将按此标准对所有专利进行分类。
如有调整意见，请指出。
```

### 1.3.3 按用户确认的分类标准标引

每篇专利给出：
- **主标签**（1个）：核心发明机制所属分类
- **次要标签**（0-2个）：辅助技术域
- **置信度**（高/中/低）

**分类汇总输出**：
```
专利总数: N 件
{分类A}: X 件
{分类B}: Y 件
...
置信度: 高X / 中Y / 低Z
```

## 1.5 全量专利技术方案输出到 Excel（精细化新增）

对**每一件专利**（含排除专利），基于其标题、摘要和 Claim 1，用一句话概括其核心技术方案，并输出为 Excel 文件，作为全量分析底表：

### 1.5.1 一句话技术方案摘要

```
格式（每件专利一行）：

```

### 1.5.2 Claim 1 要件级分解（Feature-by-Feature）

对**每一件专利**的 Claim 1 做要件分解，追加到 Excel 的列中：

| 要件字段 | 说明 |

| ... | ... |
| 创新点判断 | 真创新/改进型/常规应用/无法判断 |

> **要件分解粒度**：独立技术特征指在结构/功能/效果上可与其他特征分离的最小单元，通常来自 Claim 1 中的分句/分号段。

# Phase 2 — 价值评估与重点筛选

## 2.1 综合价值评分

三维加权评分：

| 指标 | 权重 | 评分规则 |
|------|------|---------|
| 创新性 | 50% | Phase 2初步分（从Claim 1/摘要判断创新程度），详见下方评分规则 |
| 布局国家/同族 | 30% | ≥5国=100, 3-4=70, 2=40, 1=20 |
| 法律状态 | 20% | 授权维持=100, 实审中=60, 公开=40, 驳回/放弃=0 |

综合分 = 创新性×0.50 + 布局×0.30 + 法律×0.20

**创新性初步分规则（Phase 2，全部专利）：**
- Claim 1 涉及新原理/新结构/新材料/新算法方向 → 70
- 改进型/参数优化/已知技术组合 → 50
- 已知技术的常规应用 → 30
- Claim 1 缺失或无法判断 → 40

> 创新性初步分在 Phase 3 深度检索后会被修正分覆盖（仅重点专利）。
> 修正分：真创新=80~100，替代方案=50~70，伪创新=20~40。

## 2.2 价值等级分层

| 等级 | 分数 | 标签 |
|------|------|------|
| ★★★ | ≥75 | 核心专利 |
| ★★ | 50-74 | 重要专利 |
| ★ | <50 | 外围专利 |

## 2.3 确定重点专利

- 前 30%（按综合分降序，至少 3 件）→ 进入 Phase 3 深度检索与方案提炼
- 若用户指定了重点关注专利号，优先纳入

# Phase 3 — 深度检索与方案提炼（重点专利，用于Word报告）

> **全量专利的逐件要件分解已输出到 Excel（见 1.5.2）**，此 Phase 仅针对评分 Top 30% 的重点专利，
> 做进一步 Web 检索补充和深度提炼，用于 Word 报告中的重点剖析章节。

## 3.1 Web 检索补充技术细节

对每件重点专利，使用 `exa_web_search_exa` 搜索该件专利本身的技术细节（每件 1-2 次）：

| 搜索维度 | 说明 |
|---------|------|
| 专利技术细节 | 用专利号 + 核心术语搜索，查找该专利的公开文本、同族专利等以补充 Claim 1 之外的实现细节 |

get raw texts only; then summarize in Chinese to explain each dimension below:

- **实现细节**：比 Claim 1 更具体的实现方式、参数、架构
- **产品落地**（如有）：该专利技术是否已商业化？关联产品型号？

搜索用专利号和核心术语轮流查询。

输出格式（每件专利）：
```
专利号: US12334926B2
标题: {Title}

【技术方案补充】
从检索结果中提炼 Claim 1 之外的技术细节：实现方式、具体参数、架构图描述等

【创新点确认】
基于检索结果确认：该专利的真创新/伪创新判断

【产品落地】（如有）
关联产品: {型号}
证据: {原文片段} [URL]
```

> **准确性要求**：所有内容必须有专利文本或公开文档依据，不得推断编造。

# Phase 4 — 竞争格局分析（精细化）

**所有结论必须有具体专利号作为依据，不凭印象得出结论。**

## 4.1 申请人布局分布

从分类结果统计各申请人覆盖的技术分类：

| 申请人 | 专利数 | ★★★数 | 覆盖分类 | 主攻方向 | 核心专利号 |
|--------|--------|--------|----------|----------|-----------|

**战略意图判断**：进攻（大量原创性专利）/ 防御（外围专利密集）/ 卡位（核心节点单点布局）

## 4.2 技术路线深度对比矩阵（精细化新增）

对**每个分类**（专利数 ≥ 3），按「技术目标 → 各申请人技术路径 → 方案特点 → 关键技术特征」四个层级展开对比。

**关键是：不仅要列差异，还要归纳出各家方案的「总体特征签名」**，即一眼能看出各家方案的风格和取向。

例如，伺服技术分类的对比可以总结为：
- Fujifilm方案特征：**几何精度驱动** — 精确约束伺服图案的几何尺寸比例关系
- Sony方案特征：**写入头结构创新** — 改进伺服写头的物理结构
- 两家总体差异：Fujifilm偏"算"(数学约束) vs Sony偏"造"(制造工艺)

输出格式：

```
### 分类: {分类A}

**技术目标**：{该分类下所有专利共同要解决的核心问题}

**各方案总体特征签名：**
- {公司A}方案特征：{一句话概括特征风格,如"几何精度驱动/材料配方驱动/算法优化驱动"}
  - 核心思路：{一句话核心技术思路}
  - 关键技术特征举例F1：{具体特征描述}
  - 关键技术特征举例F2：{具体特征描述}
  - 技术效果：{具体效果指标}
  - 代表专利：{专利号1} → 方案特点：{一句话}
  - 代表专利：{专利号2} → 方案特点：{一句话}

- {公司B}方案特征：{一句话概括特征风格}
  - 核心思路：{一句话核心技术思路}
```

## 核心原则

1. **只要有专利号就能跑**，缺字段自动从 Google Patents 补，不等用户手动补充
2. **Claim 1 是分析核心**：创新点边界以 Claim 1 为准，摘要仅作辅助
3. **区分真创新与伪创新**：新原理/新结构/新材料/新算法=真创新；参数微调/简单组合=如实标注
4. **趋势结论必须有专利号依据**：每条技术趋势和竞争判断引用具体专利号
5. **画布建议要有约束力**：衍生方向必须能转化为具体专利标题草案
6. **分类标准动态制定**：AI 浏览所有专利后，根据专利实际内容制定分类标准，提交用户审批后再执行分类
7. **搜索工具规范**：抓取指定 URL（如 Google Patents）用 `exa_web_fetch_exa(urls=[...], maxCharacters=100000)`；关键词搜索（如查技术细节/产品落地）用 `exa_web_search_exa(query="...", numResults=20)`。禁止使用 `web_fetch`。
8. **要件分解粒度要求**：Claim 1 分解时的「独立技术特征」指在结构/功能/效果上可与其他特征分离的最小单元，通常来自 Claim 1 中的分句/分号段
9. **Excel-Word 分工明确**：全量专利的逐件分析（一句话方案+Claim要件分解）输出到 Excel；Word 报告只聚焦整体性分析（技术分类全景/竞争格局/趋势）和重点专利的深度剖析。二者互补，不重复。
10. **话术：** When summarizing the analysis results of this patent, output it in Chinese first, and then provide the original English terms in parentheses for all English terms
11. 生成顺序：配套 Excel → Word 报告，后者引用前者的汇总数据
12. 最后 `present_files` 一次性呈现所有生成文件
