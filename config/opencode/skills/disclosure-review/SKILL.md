# 技术交底书评审 Skill

对技术交底书（发明披露文档）进行结构化质量评审，输出逐项评审意见和改进建议，
帮助发明人在提交专利代理前完成自检和质量提升。

## When to use

- 用户上传技术交底书文档（.docx / .pdf / .md / .txt）并要求评审
- 用户粘贴交底书内容并询问质量如何
- 用户要求按照专利申请标准检查交底书的合规性、完整性、清晰度

## Workflow

### Step 0: 加载评审标准

首先读取评审标准参考文件，获取完整的检查项清单：

```
view /path/to/this-skill/references/review-checklist.md
```

该文件包含 7 大评审维度、各维度下的详细检查项、评分标准和常见问题模式。
每次评审必须先加载此文件，确保评审标准一致。

### Step 1: 读取交底书内容

根据用户提供的交底书来源获取全文：

- **上传文件**：使用 file-reading skill 的策略读取文件内容（.docx 用 python-docx，
  .pdf 用 pdf-reading skill）
- **在线粘贴**：直接从对话中获取内容
- **多个文件**：如附图单独上传，一并读取

读取后，识别交底书的章节结构，确认包含哪些章节。标准章节包括：
1. 缩略语和关键术语
2. 相关技术背景 / 最接近的现有技术
3. 与本发明相关的现有技术
4. 本发明可应用的系统架构或场景
5. 本发明涉及的产品、售卖形态及侵权取证方式
6. 本发明技术实现的具体实施例（含多个实施例）
7. 本发明的主要创新点及其对应的技术效果
8. 发散思维
9. 在先公开的现有技术文献信息

### Step 2: 逐维度评审

按照 review-checklist.md 中定义的 **7 大评审维度** 依次检查：

| 维度编号 | 维度名称 | 核心关注点 |
|---------|---------|-----------|
| D1 | 整体合规性 | 合规风险、敏感用语、未上市产品信息、附图规范 |
| D2 | 术语规范性 | 技术术语准确性、缩略语完整性、前后一致性 |
| D3 | 背景技术质量 | 现有技术公开性、相关性、缺点与发明问题对应性 |
| D4 | 技术方案完整性 | 系统架构、场景描述、实施例清晰/完整/正确 |
| D5 | 商业价值与可维权性 | 产品形态、侵权取证方式、售卖形态 |
| D6 | 创新点与技术效果 | 创新点-技术效果对应关系、必要性标识、技术性语言 |
| D7 | 前瞻性与文献信息 | 发散思维、跨领域应用、IDS 文献完整性 |

对每个维度：
1. **逐项检查**每个子检查项（参见 review-checklist.md 的详细清单）
2. 对每个检查项给出 **结论**：✅ 通过 / ⚠️ 需改进 / ❌ 不通过
3. 写出 **具体问题描述**（引用交底书中的原文位置或段落）
4. 给出 **改进建议**（具体、可操作）

### Step 3: 评分

对每个维度按以下标准打分（0-10分）：

| 分数区间 | 含义 |
|---------|------|
| 9-10 | 优秀：该维度无明显问题，可直接提交代理 |
| 7-8 | 良好：有小问题但不影响专利申请质量，建议优化 |
| 5-6 | 合格：存在需要修改的问题，修改后可提交 |
| 3-4 | 较差：该维度存在较多问题，需要较大幅度修改 |
| 0-2 | 不合格：该维度严重缺失或存在致命问题 |

计算 **总体加权评分**，权重如下：
- D1 整体合规性: 10%
- D2 术语规范性: 10%
- D3 背景技术质量: 15%
- D4 技术方案完整性: 30%（最重要维度）
- D5 商业价值与可维权性: 10%
- D6 创新点与技术效果: 15%
- D7 前瞻性与文献信息: 10%

### Step 4: 生成评审报告

生成 Excel (.xlsx) 文件，包含以下 sheet：

#### Sheet 1: 评审总览
| 列 | 内容 |
|----|------|
| 交底书标题 | 从交底书中提取 |
| 评审日期 | 当前日期 |
| 总体评分 | 加权总分（保留1位小数） |
| 总体评价 | 优秀/良好/合格/较差/不合格 |
| 关键问题数 | ❌ 数量 |
| 改进建议数 | ⚠️ 数量 |
| 总体评审意见 | 2-3句话概括性评价 |

各维度评分行（D1-D7）：维度名称 | 分数 | 简要评语

#### Sheet 2: 详细评审意见
每行一个检查项：

| 维度 | 检查项 | 结论 | 问题描述 | 改进建议 | 严重程度 |
|------|--------|------|---------|---------|---------|

严重程度分为：致命（必须修改）、重要（强烈建议修改）、一般（建议优化）

#### Sheet 3: 优先改进清单
从 Sheet 2 中筛选出所有 ❌ 和 ⚠️ 项，按严重程度排序，形成可直接交给发明人的
改进行动清单：

| 优先级 | 维度 | 问题 | 改进建议 | 预计工作量 |

预计工作量分为：小（<1小时）、中（1-4小时）、大（>4小时）

**Excel 格式要求：**
- 使用 xlsx skill 的标准格式规范
- Header 行：蓝色背景 (#4472C4)，白色加粗 Arial 11pt，居中
- Body 行：Arial 10pt，自动换行，垂直居中
- 条件格式：✅ 绿色字体，⚠️ 橙色字体，❌ 红色字体
- 各列宽度合理设置，确保内容可读

保存到 `/mnt/user-data/outputs/` 并使用 `present_files` 呈现给用户。

### Step 5: 对话式总结

在呈现 Excel 后，给出简短的对话式总结：
1. 总体质量判断（一句话）
2. 最突出的 2-3 个问题（致命或重要级别的）
3. 亮点（做得好的方面）
4. 下一步建议

## 评审原则

1. **以代理师视角评审**：核心标准是"代理师能否快速、准确理解发明并撰写高质量
   专利申请文件"。技术方案描述不充分、逻辑不清晰是最常见的致命问题。

2. **技术方案完整性优先**：D4 占 30% 权重，因为实施例的清晰、完整、正确是
   交底书最核心的价值。

3. **不做技术正确性判断**：评审关注的是"写得是否清楚完整"，而不是"技术方案
   本身是否正确"。除非存在明显的逻辑矛盾或自相矛盾，不对技术方案本身做对错判断。

4. **给出具体可操作的建议**：避免泛泛而谈的建议（如"请完善"），应当指出具体
   缺失什么内容、应该补充什么。

5. **敏感内容从严**：对合规性问题（未公开技术、敏感用语、未上市产品信息等）
   采用零容忍态度，一旦发现即标记为 ❌。

## Edge cases

- **交底书缺少某些章节**：在对应维度中标记为 ❌ 并说明该章节缺失
- **交底书为英文**：同样适用所有评审标准，但术语检查项调整为检查英文术语一致性
- **用户只提供部分章节**：只评审提供的部分，在总结中说明未覆盖的维度
- **交底书附图以图片形式存在**：检查是否为黑白图、是否清晰可编辑，标注图片质量问题
